Porque é preferível usar State a Variaveis?
-A principal funcionalidade de um State é de gerenciar valores, com ele nós podemos consultar e alterar.
-O state nos permite re-renderizar um componente após sua execução, uma variável não.

Porque usar useState em Inputs?
Atrelando um use State a um Input é possível alterar o input por um evento onChange.
Ao usar useState em um formulário nós temos a possibilidade da limpeza dos inputs por meio do (Controlled Input)
Após preenchermos um Form podemos unir os States e fazer um envio dos dados para o Backend.


USE REDUCER
-Assim como o useState o Reducer gerencia valores.
-Entretanto, com o reducer é possível executar uma função ao açterar um valor
